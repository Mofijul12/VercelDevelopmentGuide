MongoDB Atlas Configuration
Step 1: Create Database User
  1. Go to cloud.mongodb.com and sign in
  2. Navigate to Database Access under Security section
  3. Click Add New Database User
  4. Choose Password authentication method
  5. Enter username (example: myappuser)
  6. Click Autogenerate Secure Password
  7. IMMEDIATELY copy and save the password
  8. Set privileges to Read and write to any database
  9. Click Add User
Critical: Save your password immediately! You cannot retrieve it later. If lost, you'll need to reset it.
Step 2: Configure Network Access
  1. Go to Network Access under Security
  2. Click Add IP Address
  3. Click Allow Access from Anywhere
  4. This adds IP: 0.0.0.0/0
  5. Add comment: "Vercel Deployment"
  6. Click Confirm
  7. Wait 2-3 minutes for changes to propagate
Why allow all IPs? Vercel uses dynamic serverless functions with changing IPs, so we need to allow access from any IP address.
Step 3: Get Connection String
  1. Go to Database section
  2. Click Connect button on your cluster
  3. Choose Connect your application
  4. Select Driver: Node.js, Version: 5.5 or later
  5. Copy the connection string
  6. Replace <password> with your actual password
  7. Add database name before ? symbol
mongodb+srv://myappuser:MyPassword123@cluster0.abc123.mongodb.net/myDatabase?retryWrites=true&w=majority
Format breakdown:
• Username: myappuser
• Password: MyPassword123
• Cluster: cluster0.abc123.mongodb.net
• Database: myDatabase

Step 2: Create server.js

Main backend file with CRUD operations

const express = require('express');
const mongoose = require('mongoose');
const cors = require('cors');
require('dotenv').config();

const app = express();

// Middleware
app.use(cors());
app.use(express.json());

// MongoDB Connection
mongoose.connect(process.env.MONGODB_URI)
.then(() => console.log('MongoDB Connected'))
.catch(err => console.log('MongoDB Error:', err));

// Schema & Model
const itemSchema = new mongoose.Schema({
  name: { type: String, required: true },
  description: String,
  createdAt: { type: Date, default: Date.now }
});

const Item = mongoose.model('Item', itemSchema);

// CREATE
app.post('/api/items', async (req, res) => {
  try {
    const newItem = new Item(req.body);
    const savedItem = await newItem.save();
    res.status(201).json(savedItem);
  } catch (error) {
    res.status(400).json({ message: error.message });
  }
});

// READ ALL
app.get('/api/items', async (req, res) => {
  try {
    const items = await Item.find().sort({ createdAt: -1 });
    res.json(items);
  } catch (error) {
    res.status(500).json({ message: error.message });
  }
});

// READ ONE
app.get('/api/items/:id', async (req, res) => {
  try {
    const item = await Item.findById(req.params.id);
    if (!item) return res.status(404).json({ message: 'Not found' });
    res.json(item);
  } catch (error) {
    res.status(500).json({ message: error.message });
  }
});

// UPDATE
app.put('/api/items/:id', async (req, res) => {
  try {
    const updatedItem = await Item.findByIdAndUpdate(
      req.params.id, req.body, { new: true }
    );
    if (!updatedItem) return res.status(404).json({ message: 'Not found' });
    res.json(updatedItem);
  } catch (error) {
    res.status(400).json({ message: error.message });
  }
});

// DELETE
app.delete('/api/items/:id', async (req, res) => {
  try {
    const deletedItem = await Item.findByIdAndDelete(req.params.id);
    if (!deletedItem) return res.status(404).json({ message: 'Not found' });
    res.json({ message: 'Deleted successfully' });
  } catch (error) {
    res.status(500).json({ message: error.message });
  }
});

// Health check
app.get('/', (req, res) => {
  res.json({ message: 'API is running' });
});

const PORT = process.env.PORT || 5000;
app.listen(PORT, () => console.log(`Server on port ${PORT}`));

Final Configuration & Testing
Understanding CORS
What is CORS?
CORS (Cross-Origin Resource Sharing) is a security feature that prevents websites from making requests to different domains. Since your frontend and backend are on different Vercel URLs, you need to explicitly allow this.

Without CORS: Browser blocks requests from frontend to backend

With CORS: Backend tells browser "it's okay to accept requests from this frontend"

Step 1: Update Backend CORS

In your backend server.js, find this line:

app.use(cors());

Replace it with:

app.use(cors({
  origin: [
    'http://localhost:3000',                    // Local development
    'https://my-app-frontend-xyz.vercel.app'    // Production frontend
  ],
  credentials: true
}));
Replace https://my-app-frontend-xyz.vercel.app with your actual frontend URL!
Step 2: Push CORS Changes
cd my-app-backend

git commit -m "Update CORS for production frontend"
git push

Vercel will automatically redeploy your backend (takes 1-2 minutes)

Troubleshooting & Maintenance
Common Issue 1: CORS Error

Symptoms:

  • Error in browser console: "blocked by CORS policy"
  • Frontend can't fetch data from backend
  • Network tab shows request failed

Solutions:

  1. Verify frontend URL is in backend CORS configuration
  2. Check for typos in URL (https vs http, trailing slash)
  3. Ensure both URLs are included: localhost AND production
  4. Redeploy backend after CORS changes
  5. Clear browser cache and hard reload (Ctrl+Shift+R)
Common Issue 2: MongoDB Connection Timeout

Symptoms:

  • "Operation timed out after 10000ms"
  • "querySrv ENOTFOUND"
  • Backend returns 500 errors

Solutions:

  1. Check Network Access in MongoDB Atlas has 0.0.0.0/0
  2. Verify MONGODB_URI in Vercel environment variables
  3. Ensure connection string includes database name
  4. Check password has no special characters or is URL-encoded
  5. Wait 2-3 minutes after whitelist changes
  6. Redeploy backend in Vercel
Common Issue 3: Module Not Found

Symptoms:

  • "Cannot find module 'express'"
  • Build fails on Vercel

Solutions:

  1. Verify all dependencies in package.json
  2. Check package-lock.json is committed to GitHub
  3. Delete node_modules and package-lock.json locally
  4. Run npm install again
  5. Commit and push changes
  6. Redeploy on Vercel
Common Issue 4: Environment Variables Not Working

Symptoms:

  • Variables showing as undefined
  • App works locally but not in production

Solutions:

  1. Frontend: Must start with REACT_APP_
  2. Check spelling and casing (case-sensitive)
  3. Restart dev server after changing .env
  4. Redeploy after adding/changing Vercel env vars
  5. Verify env vars are set for correct environments
Common Issue 5: 500 Internal Server Error

Symptoms:

  • API requests return 500 status
  • App crashes on certain operations

Solutions:

  1. Check Vercel logs: Deployments → Click deployment → Logs
  2. Look for specific error messages
  3. Common causes: MongoDB connection, validation errors, syntax errors
  4. Test the same request locally to reproduce
  5. Add console.log statements for debugging
